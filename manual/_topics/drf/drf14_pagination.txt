14. Pagination (Пагинация)


1. PageNumberPagination
    Пагинация принимает номер страницы с одним номером в параметрах запроса.
    - django_paginator_class
        Класс Django Paginator. По умолчанию django.core.paginator.Paginator.
    - page_size
        Размер страницы. По умолчанию PAGE_SIZE.
    - page_query_param
        Имя параметра запроса для управления пагинацией.
    - page_size_query_param
        Имя параметра запроса, который позволяет клиенту устанавливать размер 
        страницы на основе каждого запроса.
    - max_page_size
        Максимально допустимый размер запрашиваемой страницы. Действителен, 
        если установлен page_size_query_param.
    - last_page_strings
        Значения, которые могут быть испрользованы с page_query_param для 
        запроса последней страницы в наборе. По умолчанию ('last',).
    - template
        Имя шаблона для отображения элементов управления пагинацией в 
        просматриваемом API. По умолчанию 
        'rest_framework/pagination/numbers.html'.
LimitOffsetPagination
    Пагинация повторяет синтаксис, используемый при поиске нескольких записей в 
    БД. Клиент включает в себя 'limit' и 'offset'. Лимит указывает на 
    максимальное количество возвращаемых элементов. Смещение указывает 
    начальную позицию запроса.
    - default_limit
        Предел, который следует использовать, если он не указан клиентом в 
        параметре запроса. По умолчанию PAGE_SIZE.
    - limit_query_param
        Имя параметра запроса 'limit'. По умолчанию 'limit'.
    - offset_query_param
        Имя параметра запроса 'offset'. По умолчанию 'offset'.
    - max_limit
        Максимально допустимый лимит, который может быть запрошен клиентом. По 
        умолчанию None.
    - template
        Имя шаблона для отображения элементов управления пагинацией в 
        просматриваемом API. По умолчанию 
        'rest_framework/pagination/numbers.html'.
CursorPagination
    Пагинация представляет только элементы управления перемоткой вперед и назад 
    и не позволяет клиенту переходить к произвольным позициям. Пагинация 
    требует наличия уникального, неизменного порядка следования элементов в 
    наборе результатов. Обычно это временная метка создания записей, так как 
    она представляет собой последовательный порядок для постраничного просмотра.
    - page_size
        Размер страницы. По умолчанию PAGE_SIZE.
    - cursor_query_param
        Имя параметра запроса 'cursor'. По умолчанию 'cursor'.
    - ordering
        строка или список строк, указывающих на поле, к которому будет 
        применяться пагинация на основе курсора. Наример, ordering='slug'. 
        По умолчанию 'created'.
    - template
        Имя шаблона для отображения элементов управления пагинацией в 
        просматриваемом API. По умолчанию 
        'rest_framework/pagination/numbers.html'.

Для создания пользовательской пагинации, необходимо наследовать BasePagination, 
переопределить методы paginate_queryset(self, queryset, request, view=None) и 
get_paginated(self, data). paginate_queryset передается начальному кверисету и 
должен возвращать итерируемый обьект. Содержит только данные запрашиваемой 
страницы. get_paginated_response передается сериализованным данным страницы и 
должен возвращать экземпляр Response.
